## 会话 1 · 初始化:第 0 步 loss 与 tanh 饱和

### 1.1 第 0 步 loss 应该等于多少(课前问题 1)

**跑之前就能算出来**:初始化时模型对 27 类应该一无所知 → 均匀分布 → loss = −log(1/27) ≈ **3.30**。

实际却是 **27**。诊断:初始 logits 数值巨大(随手 randn 的 W2/b2 所致),softmax 输出极端自信,而方向纯属随机——**confidently wrong**。loss 曲线呈 hockey stick:前几千步全浪费在把 logits 压回 0 附近,做的是"消除错误自信"这种本不必要的功。

修复:让初始 logits ≈ 0(此时 softmax 自动输出均匀分布):

```python
W2 = torch.randn((n_hidden, 27)) * 0.01   # 别精确为 0, 留点熵打破对称
b2 = torch.zeros(27)
# 第 0 步 loss → 3.30 ✅
```

**从此养成的反射**:训练任何分类模型,先心算瞎猜 loss(−log(1/C)),对照第 0 步实际值。对不上=初始化有病。这招零成本,能拦住大量 bug。

### 1.2 tanh 饱和:被杀死的梯度(课前问题 2)

修好输出层后看隐层:`plt.hist(h.view(-1))` 显示 tanh 输出**大量堆积在 ±1**。为什么这是灾难——回忆 micrograd 里 tanh 的本地导数:

```
∂tanh/∂x = 1 − t²    →    t = ±1 时本地导数 = 0
```

tanh 是个软门:饱和 = 门关死,**无论上游梯度多大,乘 0 之后什么都传不过去**。这一层的 W1/b1 以及更浅层全部失学。

可视化(白图):`plt.imshow(torch.abs(h) > 0.99)`,白点 = 饱和。若某个神经元**对所有样本**都饱和(一整列全白)= **死神经元**,永远不会再学习。同样的病在 ReLU 上叫 dead ReLU(负半轴平坦区),不仅初始化会造成,训练中步子太大把神经元"踢出"活跃区也会——永久性脑损伤。

病因:进 tanh 的预激活 `emb @ W1 + b1` 方差太大,把 tanh 推到平坦区。

### 1.3 从"拍脑袋乘小数"到 Kaiming 初始化(课前问题 3)

临时药方:`W1 *= 0.2`——有效,但 0.2 哪来的?推导增益:

若 `y = x @ W`,x 各维独立零均值单位方差,W ~ N(0, σ²),则 `Var(y) = fan_in · σ²`。要 Var(y) = 1,须:

```
σ = 1 / √fan_in
```

再考虑激活函数把方差压掉一部分,补一个增益 gain:**tanh 的 gain = 5/3**(ReLU 是 √2,原因:砍掉一半分布)。合起来就是 **Kaiming 初始化**(He et al. 2015):

```python
W1 = torch.randn((fan_in, n_hidden)) * (5/3) / fan_in**0.5
# PyTorch 等价: torch.nn.init.kaiming_normal_(W1, nonlinearity='tanh')
```

SP 翻译:每级插入 1/√fan_in 的定标,使级联系统各级输出功率守恒——这就是全部原理,没有玄学。

现代注脚:有了 BatchNorm、残差连接、Adam 之后,初始化精度没那么生死攸关了,但"第 0 步 loss 检查"和"方差守恒直觉"永不过时。

## 会话 2 · BatchNorm

### 2.1 思想:既然想要单位高斯,就直接标准化(课前问题 1)

Kaiming 初始化是**间接**手段:精心定标权重,祈祷预激活是单位高斯。BatchNorm(2015)说:**别祈祷了,直接算出当前 batch 的均值方差,标准化掉**:

```python
# 训练时, 对预激活 hpreact (B, n_hidden):
bnmeani = hpreact.mean(0, keepdim=True)          # (1, n_hidden) 按 batch 维求均值
bnstdi  = hpreact.std(0, keepdim=True)
hpreact = bngain * (hpreact - bnmeani) / (bnstdi + 1e-5) + bnbias
h = torch.tanh(hpreact)
```

BatchNorm 想要的统计性质:**预激活(进 tanh 之前的量)在 batch 维上近似零均值、单位方差**——恰好落在 tanh 的活跃区。

为什么标准化之后还要 gain(γ)和 bias(β):我们只想**在初始时**是单位高斯,不想永远锁死——网络可能需要更宽/偏移的分布来表达。γ 初始化为 1、β 为 0(初始 = 纯标准化),之后作为**可训练参数**让梯度决定分布该长什么样。没有 γ/β,BatchNorm 会削减模型的表达能力;有了它们,表达能力无损,只是"默认状态"变健康了。

位置:线性层之后、非线性之前。ε 防除零。

### 2.2 batch 耦合:bug 还是 feature(课前问题 2)

BatchNorm 有个诡异的副作用:均值方差来自当前 batch,**每个样本的输出取决于它和谁同批**。

- **Feature 的一面**:相当于给每个样本注入依赖同伴的噪声(数据增强式抖动),实测是有效的正则化;这也是 BN 网络往往不太过拟合的原因之一。
- **Bug 的一面**:训练/推理行为不一致、batch size 依赖、分布式训练要同步统计量、样本间信息泄漏(检索/对比学习中是真事故)。工程史上 BN 是 bug 重灾区。

所以业界的长期走向是**保留归一化、去掉 batch 耦合**:LayerNorm(Transformer 标配,GPT 视频见)、GroupNorm 等——归一化的轴从"batch 维"换到"特征维",样本之间互不干扰。

### 2.3 推理时没有 batch:running statistics(课前问题 3)

推理时可能一次只进一个样本,均值方差从哪来?答案:**训练过程中用指数滑动平均(EMA)顺手记下全体数据的统计量**:

```python
with torch.no_grad():
    bnmean_running = 0.999 * bnmean_running + 0.001 * bnmeani
    bnstd_running  = 0.999 * bnstd_running  + 0.001 * bnstdi
# 推理时: hpreact = bngain * (hpreact - bnmean_running) / (bnstd_running + 1e-5) + bnbias
```

momentum(此处 0.001)是又一个超参:batch 大、任务稳可以取大些(PyTorch 默认 0.1);batch 小则统计量噪声大,要取小。另一条路是训练完后单独跑一遍全数据校准统计量,EMA 只是把这步摊进了训练。这就是 PyTorch `model.eval()` 切换的其中一半含义(另一半是 dropout)。

### 2.4 一个被 BN 变多余的参数:前置 bias

```python
hpreact = embcat @ W1 + b1        # b1 加上去...
hpreact = hpreact - hpreact.mean(0)  # ...又被减均值精确抵消
```

BN 的减均值操作把线性层的 bias **完全吃掉**(b1 是常数,对每个样本相同,减 batch 均值后不留痕迹)。它不报错、梯度也在动,只是纯浪费,而且它的角色已由 β 接管。规矩:**线性/卷积层后接 BN 时,设 bias=False**。这是"看懂机制才能删对参数"的典型例子。

## 会话 3 · 诊断仪表盘(本视频真正的遗产)

先把代码 PyTorch 化(Linear / BatchNorm1d / Tanh 类,接口对齐 `torch.nn`),搭一个 **6 层 MLP**,然后建仪表盘。

### 3.1 四张图及其健康形态(课前问题 1)

**图 1 · 前向:各层激活分布**(tanh 输出逐层画直方图,记 std 与饱和率 |t|>0.97)

- 健康:各层分布形状**大体一致**,std 稳定(tanh 网约 0.65),饱和率个位数百分比;
- 逐层**收缩**(std 递减,分布向 0 挤):增益不足,信号衰减,深处的层收不到信息;
- 逐层**发散**(饱和率递增):增益过大,信号削顶。
- 实验:gain 取 1(偏小,收缩)、3(偏大,饱和)、5/3(恰好稳定)——Kaiming gain 的数值意义肉眼可见。

**图 2 · 反向:各层梯度分布**(对激活的梯度逐层直方图)

- 健康:各层梯度分布也应大体一致;
- 逐层递减(越浅越小)= 梯度消失,浅层失学;逐层递增 = 梯度爆炸。前向与反向是同一枚硬币的两面。

**图 3 · 参数梯度/数据比**(每个权重张量的 grad.std / data.std)

- 量级应大体相近;某层比值异常大 = 该层参数被猛烈改动,常见于输出层或失配的初始化。

**图 4 ★ · update/param 比值随时间**(最有用的一张)

```python
with torch.no_grad():
    ud.append([(lr * p.grad.std() / p.data.std()).log10().item() for p in parameters])
```

- 健康:log10 比值 ≈ **−3**,即每步更新幅度约为参数尺度的 **1/1000**(课前问题 2);
- 与 lr 的关系:比值 ∝ lr,是**比 loss 曲线更直接的 lr 标定器**——远低于 −3 = lr 太小(训不动),远高于 −3 = lr 太大(要出事);
- 各参数张量的曲线应大致抱团;某条明显离群 = 那一层有问题。

### 3.2 为什么没 BN 时如此脆弱、有 BN 后如此皮实

同一个 6 层网做对照:

- 无 BN + 错误 gain:图 1 立刻显形(收缩或饱和雪崩),必须精确使用 5/3;
- 加 BN:gain 随便设、W1 尺度随便乘,四张图照样健康——**BN 把"逐层统计健康"从初始化的责任变成了架构的内置保证**。代价就是 2.2 的 batch 耦合。
- 有趣的实验:BN 网络里把 lr 加大 10 倍也不炸——归一化天然抑制了预激活的尺度漂移。

## Chapter 3 复习卡

1. **两个检查点从此常驻**:第 0 步 loss = −log(1/C);update/param ≈ 1e-3。
2. **一条方差守恒律**:σ = gain/√fan_in(tanh gain 5/3,ReLU √2)——初始化不是玄学,是功率定标。
3. **饱和三段论**:预激活方差大 → 激活贴 ±1 → 本地导数 1−t²≈0 → 梯度死。死神经元 = 对所有样本恒饱和。
4. **BatchNorm 一句话**:训练时按 batch 标准化预激活 + 可学习 γ/β 保表达力 + running stats 供推理;代价是 batch 耦合(正则是甜头,bug 是账单),后继者是 LayerNorm。
5. **BN 后线性层 bias=False**——能说出为什么。
6. 四张图 = 前向健康、反向健康、参数改动幅度、更新步长标定。

**易错点清单**:
- BatchNorm 训练/推理没切换(推理还在用 batch 统计量,batch=1 时直接 NaN)
- running stats 更新忘了包 `torch.no_grad()`(白白建图,甚至污染梯度)
- std 的归一化轴写错(该沿 batch 维 dim=0,写成 dim=1)
- 对照实验时只看 loss 不看图(本集的全部意义就在图里)
- W2 初始化精确置 0(建议留 0.01 的小噪声)
