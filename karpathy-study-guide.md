## Chapter 1|micrograd(视频约 2.5h → 3 个会话)

### 会话 1:导数直觉 → Value 类与手动反向(≈90min)
范围:开头至"手动做完一个小表达式的 backprop"(章节:intro / derivative / Value object 附近)。
课前问题:
1. 只用 (f(x+h)-f(x))/h 这一个工具,如何得到一个多输入表达式对每个输入的敏感度?
2. 链式法则在"计算图"上对应什么操作?局部梯度指什么?
3. 为什么每个 Value 节点要同时存 data 和 grad 两个字段?
⏹ 手动(不用 backward())算对一个小表达式所有节点的梯度。

### 会话 2:backward() 自动化 → tanh 与梯度累积(≈90min)
范围:拓扑排序实现 backward,至 tanh 神经元与"bug: 梯度累积"段落。
课前问题:
1. 为什么反向传播必须按拓扑序?乱序会错在哪?
2. 一个节点被使用两次(如 b = a + a),梯度应该怎么处理?`+=` 和 `=` 的差别会造成什么 bug?
3. tanh 可以作为一个原子操作实现,也可以拆成 exp/加减乘除的复合——两种实现的梯度会一致吗?为什么?
⏹ backward() 跑通,与会话 1 的手算结果一致。

### 会话 3:MLP + 训练循环 + 对照 PyTorch(≈90min)
范围:Neuron/Layer/MLP 类,loss,训练循环,至与 PyTorch 对照收尾。
课前问题:
1. 一次完整的训练迭代由哪四步构成?哪一步最容易被忘掉、忘掉后症状是什么?(提示:zero_grad)
2. 学习率过大/过小分别在 loss 曲线上长什么样?
3. micrograd 的 Value 和 PyTorch 的 tensor,最本质的对应关系是什么?
⏹ 二层 MLP 在小数据集上 loss 稳定下降;PyTorch 对照段跑通。

## Chapter 2|makemore #1 bigram(视频约 2h → 2 个会话)

### 会话 2:神经网络版 bigram + 等价性(≈100min)

## Chapter 3|makemore #3 激活/梯度/BatchNorm(视频约 2h → 3 个会话)

### 会话 1:初始化问题(≈60min)
课前问题:
1. 训练第 0 步的 loss 应该约等于多少?(能不能在跑之前算出来?)如果实际远高于它,说明什么?
2. tanh 输出全在 ±1 附近意味着什么?为什么这会"杀死"梯度?
3. 把权重初始化简单乘个小系数,为什么有效?系数怎么定才不是拍脑袋?(引出 Kaiming)
⏹ 复现"初始 loss 过高→修复"的全过程,画出 tanh 激活直方图修复前后的对比。

### 会话 2:BatchNorm(≈75min)
课前问题:
1. BatchNorm 想让隐层的"预激活"满足什么统计性质?为什么标准化之后还要 gain 和 bias?
2. BatchNorm 让每个样本的输出依赖同 batch 的其他样本——这是 bug 还是 feature?
3. 推理时没有 batch,均值方差从哪来?
⏹ 手写 BatchNorm 层并训练通过;能说出它放在线性层之后为什么 bias 变得多余。

### 会话 3:诊断仪表盘(≈75min)
课前问题:
1. 健康的训练里,各层激活分布、梯度分布应该长什么样?逐层递减/递增说明什么?
2. update/param 比值多大算健康?这个数和学习率是什么关系?
3. 如果只允许看三张图判断训练是否健康,你选哪三张?
⏹ 复现视频里的四类统计图,并存为你以后所有项目可复用的诊断代码片段(这是本视频的真正遗产)。

## Chapter 4|makemore #4 backprop ninja(视频约 2h → 3 个会话)
> 本集形态特殊:一个大 exercise,逐张量手写梯度再与 autograd 对答案。切分按 exercise 的自然分段。

### 会话 2:线性层 + tanh + embedding 反向(≈90min)

3. batch 维度上梯度为什么是"求和"而不是"平均"?什么时候会变成平均?

### 会话 3:BatchNorm 反向 + 整合(≈90min)

2. 手推出的 BatchNorm 梯度公式,哪些项对应"去均值",哪些对应"去相关"?
3. 做完这一切,你获得的能力用一句话概括是什么?(自己写答案,收工时修订)
⏹ 全视频 exercises 完成;★ 隔日闭卷:手推 softmax+CE 合并梯度。

## 使用备注
- 会话间隔 1–3 天最佳;同一视频不建议一天连吃三个会话(违背间隔效应)。
- 课前问题答不上是**正常且正确**的状态——它们的作用是标记"待填的坑";收工自测时仍答不上,才说明该回看。
- 冲刺周加速的方式是"多排会话",不是"跳过收工状态检查"。
